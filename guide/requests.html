# requests 类

requests 类提供了发送不同类型的 HTTP 请求的方法，并返回响应对象。

# 方法

# get($url, $data=[])

发送 GET 请求并返回响应对象。

参数：

  • $url (string) - 请求的 URL。
  • $data (array) - 请求的查询参数，默认为空数组。

返回值：

  • (Response) - 响应对象。

# post($url, $data=[])

发送 POST 请求并返回响应对象。

参数：

  • $url (string) - 请求的 URL。
  • $data (array) - 请求的表单数据，默认为空数组。

返回值：

  • (Response) - 响应对象。

# 可选参数

  • ->cookie([]) (array) - 请求的 Cookie，默认为空数组。
  • ->header([]) (array) - 请求的头部信息，默认为空数组。

# 其他 HTTP 请求方法

requests 类还提供了以下其他 HTTP 请求方法：put(), delete(), patch(), head(), connect(), trace(), options()。这些方法的参数和返回值与上述方法相同。其中put方法仅支持二进制的$data

# 响应对象 (Response)

Response 对象封装了 HTTP 响应的相关信息和方法。

# 方法

# json()

将响应体解析为 JSON 格式，并返回解析后的数组或字符串。

返回值：

  • (mixed) - 解析后的 JSON 数据。
# text()

返回原始的响应体内容。

返回值：

  • (string) - 原始的响应体内容。
# code()

返回响应的 HTTP 状态码。

返回值：

  • (int) - 响应的 HTTP 状态码。
# headers()

返回响应的头部信息数组。

返回值：

  • (array) - 响应的头部信息数组。

# 使用示例

此实例已自动创建并初始化到全局常量requests，无需在插件内初始化 若需要重复初始化，使用代码

include_once($_SERVER['DOCUMENT_ROOT'].'/services/requests.php');

$requests = new requests();

<?php

// 发送 GET 请求并获取响应的文本内容
$response = requests->get("https://example.com");
echo $response->text(); // 输出响应的文本内容

// 发送 GET 请求并将响应解析为 JSON 格式
$response = requests->get("https://example.com/api");
$data = $response->json(); // 解析 JSON 响应
print_r($data); // 输出解析后的 JSON 数据

// 发送 GET 请求并获取响应的状态码
$response = requests->get("https://example.com");
$code = $response->code(); // 获取响应的状态码
echo $code; // 输出响应的状态码

// 发送 GET 请求并获取响应的头部信息
$response = requests->get("https://example.com");
$headers = $response->headers(); // 获取响应的头部信息
print_r($headers); // 输出响应的头部信息
?>
